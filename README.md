# 日本語 音声文字起こしツール

`.wav` 音声ファイルを**ローカル環境**で日本語に文字起こしする GUI ツールです。
音声データは外部サーバへ送信されず、すべてこの PC 内で処理されます。

## 主な機能

- 日本語の高精度文字起こし（Whisper large-v3）
- **進捗バーと、文字起こしのリアルタイム途中経過表示**
- **複数ファイルの一括処理**（GUI で複数 `.wav` を選択し、順番に文字起こし）
- **CLI モードのバッチ処理**（`INPUT` フォルダ内の `.wav` をまとめて文字起こし）
- **話者分離**（誰が話したかを推定し、話者ごとにラベル付け）
- 結果の画面表示（全文 / タイムスタンプ付き / 話者別）
- ダウンロード（テキスト `.txt` / 話者分離テキスト `.txt` / 字幕 `.srt`）
- **LLM(Amazon Bedrock)による問い合わせ分析JSON出力**（話者分離された文字起こしを
  コンタクトセンター向けの構造化JSONへ整形。GUI / CLI 双方に対応）

## システムアーキテクチャ

| 層 | 採用技術 | 役割 |
|----|----------|------|
| 文字起こしエンジン | **faster-whisper（Whisper large-v3）** | 日本語を高精度で認識。CTranslate2 によりローカルで高速・省メモリに動作 |
| 話者分離 | **SpeechBrain（ECAPA-TDNN）＋ scikit-learn** | 各発話区間の話者埋め込みを計算しクラスタリングして話者を割り当て |
| バックエンド | **Flask（Python）** | アップロードを受け取り、文字起こしをバックグラウンドのジョブとして実行。進捗をAPIで提供 |
| フロントエンド | **HTML / JavaScript** | アップロード・進捗(残り時間)表示・結果表示・ダウンロードの GUI |

- 音声デコードは faster-whisper 内蔵の `av`（PyAV）が行うため、**ffmpeg の別途インストールは不要**です。
- 既定は CPU 動作。NVIDIA GPU 環境では `WHISPER_DEVICE=cuda` で高速化できます。

### 進捗・リアルタイム表示の仕組み
文字起こしはバックグラウンドのジョブとして実行され、フロントエンドが
`/progress/<job_id>` を定期的に取得します。Whisper はセグメント単位で
処理が進むため、処理位置（進捗バー）と、完了したセグメントのテキストを
リアルタイムに画面へ反映します。

## 必要環境

- Windows 10 / 11
- Python 3.9 以上（3.11 で動作確認）

## Python のインストール（はじめての方へ）

このツールを動かすには、まず **Python** というソフトを 1 回だけインストールする必要があります。
すでに Python が入っている場合は、このまま次の「セットアップ」へ進んでください。
入っているか分からない場合は、下の手順どおりにインストールして問題ありません。

> ⚠️ いちばん大事なポイント：手順 ③ の **「Add python.exe to PATH」のチェックを必ず入れる**ことです。
> ここを忘れると、後のセットアップで「Python が見つかりません」というエラーになります。

### 手順

1. **ダウンロードページを開く**
   下のリンクをクリックします（ブラウザが開きます）。
   👉 https://www.python.org/downloads/windows/
   ページ上部の黄色いボタン **「Download Python 3.x.x」** をクリックすると、
   インストーラ（`python-3.x.x-amd64.exe` という名前のファイル）がダウンロードされます。

2. **インストーラを起動する**
   ダウンロードしたファイル（多くの場合「ダウンロード」フォルダにあります）を
   **ダブルクリック**します。

3. **【最重要】PATH のチェックを入れる**
   最初の画面の一番下にある
   ☑ **「Add python.exe to PATH」**（または「Add Python to PATH」）に
   **必ずチェックを入れてください。**

4. **インストールする**
   画面中央の **「Install Now」** をクリックします。
   「このアプリがデバイスに変更を加えることを許可しますか？」と聞かれたら **「はい」** を選びます。
   しばらく待つとインストールが完了します。**「Close」** で閉じてください。

5. **（任意）正しく入ったか確認する**
   キーボードの **Windows キー** を押し、`cmd` と入力して
   **「コマンド プロンプト」** を開きます。黒い画面に

   ```
   python --version
   ```

   と入力して **Enter** を押し、`Python 3.x.x` のように表示されれば成功です。

> 💡 **もし手順 ③ のチェックを忘れてしまったら：**
> いったん Python をアンインストール（Windows の「設定」→「アプリ」から削除）し、
> もう一度インストーラを起動して、今度は **必ずチェックを入れて**インストールし直してください。
> （または、再度インストーラを起動して「Modify」から修正することもできます。）

## セットアップ（初回のみ）

Python のインストールが終わったら、エクスプローラで `setup.bat` をダブルクリックしてください。
仮想環境の作成と、PyTorch(CPU版)・各種ライブラリのインストールが自動で行われます。

> 「WindowsによってPCが保護されました」という青い画面が出た場合は、
> 「詳細情報」→「実行」の順にクリックすると進められます（自作のバッチファイルのため表示されることがあります）。

> 話者分離のため PyTorch / SpeechBrain を使用します。初回セットアップでは
> 数百 MB のダウンロードが発生します。

## 起動

`start.bat` をダブルクリックします。自動的に既定ブラウザで
`http://127.0.0.1:5000` が開きます。

1. `.wav` ファイルをドラッグ＆ドロップ（またはクリックで選択）。**複数ファイルをまとめて選択でき、順番に処理されます**
2. モデルを選択（既定: large-v3＝最高精度）
3. 必要なら「**話者分離を行う**」にチェック（話者の人数が分かれば入力、空欄なら自動推定）
4. 「文字起こし開始」をクリック → **進捗バーと途中経過**が表示されます
5. 結果を画面で確認し、**テキスト / 話者分離テキスト / 字幕(.srt)** でダウンロード

> 複数ファイルを選択した場合は、選択一覧に各ファイルの進行状況（待機 / 処理中 / 完了 / 失敗）が表示され、
> 上から順に1ファイルずつ処理されます。完了後は結果一覧から各ファイルの結果を切り替えて確認でき、
> 「**全ファイルの全文をまとめてDL**」で全結果を1つのテキストにまとめて保存できます。

## CLI モード（フォルダ一括処理）

GUI を使わず、**フォルダ内の `.wav` をまとめて文字起こし**するモードです。
大量のファイルを夜間に処理したいときなどに便利です。

`transcribe_folder.bat` をダブルクリックすると、`INPUT` フォルダ内の `.wav` を順番に
文字起こしし、結果を `OUTPUT` フォルダに書き出します（`INPUT` フォルダが無ければ自動作成します）。
**このバッチは既定で話者分離つき（`--diarize`）で実行し、字幕(.srt)は出力しません（`--no-srt`）。**
（話者の人数は自動推定）。人数が分かっている場合は `transcribe_folder.bat --num-speakers 2` のように
指定でき、話者分離が不要な場合や字幕(.srt)も欲しい場合は後述の `cli.py` を直接実行してください。

コマンドラインから直接実行する場合は次のとおりです。

```bat
venv\Scripts\python.exe cli.py                    rem INPUT 配下の .wav を OUTPUT へ
venv\Scripts\python.exe cli.py --diarize          rem 話者分離つき
venv\Scripts\python.exe cli.py -i 録音 -o 結果 -m medium
venv\Scripts\python.exe cli.py --recursive        rem サブフォルダも対象にする
```

`transcribe_folder.bat` に渡した引数はそのまま `cli.py` へ渡されます
（例: `transcribe_folder.bat --diarize --model medium`）。

| オプション | 既定 | 説明 |
|------------|------|------|
| `-i, --input` | `INPUT` | 入力フォルダ |
| `-o, --output` | `OUTPUT` | 出力フォルダ |
| `-m, --model` | `large-v3` | Whisper モデル |
| `--diarize` | （無効） | 話者分離を行う |
| `--num-speakers` | 自動推定 | 話者の人数を指定 |
| `-r, --recursive` | （無効） | サブフォルダ内の `.wav` も対象にする |
| `--overwrite` | （無効） | 既に出力がある場合も上書き（既定はスキップ） |
| `--no-srt` | （無効） | 字幕(.srt)を出力しない |
| `--analyze` | （無効） | LLM(Bedrock)で問い合わせ分析JSONを出力する |

出力ファイル（入力ファイル名を基準）:

- `<名前>.txt` … 全文テキスト
- `<名前>.srt` … 字幕（タイムスタンプ付き）
- `<名前>.speakers.txt` … 話者別テキスト（`--diarize` 指定時のみ）
- `<名前>.analysis.json` … LLM整形済みの問い合わせ分析JSON（`--analyze` 指定時のみ）

> **初回実行時**は選択したモデル（large-v3 は約 3GB、話者分離モデルは約 80MB）が
> 自動ダウンロードされます。2 回目以降はキャッシュされ高速に起動します。

## モデルの選び方

| モデル | 精度 | 速度（CPU） | 用途 |
|--------|------|-------------|------|
| large-v3 | ★★★ | 遅い | 精度最優先（既定・推奨） |
| medium | ★★ | 普通 | バランス重視 |
| small | ★ | 速い | 下書き・短時間で確認したいとき |

CPU のみの環境で長い音声を large-v3 で処理すると時間がかかります。
急ぐ場合は medium / small を選択してください。

## 話者分離について

- 各発話区間の声の特徴（話者埋め込み）を計算し、似た声どうしをまとめて
  「話者1」「話者2」…と割り当てます。
- **人数が分かっている場合は「話者の人数」を入力すると最も正確**です。
  空欄の場合は自動で人数を推定します（推定はあくまで目安です）。
- 精度は音質に依存します。特に **8kHz の電話録音など帯域の狭い音声**では
  声の判別が難しく、分離精度が下がることがあります。その場合は人数を明示
  指定してください。自動推定の感度は環境変数 `DIARIZE_THRESHOLD` で調整できます
  （値を小さくすると話者が増えやすく、大きくすると減りやすくなります）。

## LLM(Amazon Bedrock)による問い合わせ分析

話者分離された文字起こしデータを、Amazon Bedrock 上の Claude に渡して
コンタクトセンター向けの**構造化JSON**（問い合わせ単位の要約・カテゴリ・
業務影響度・解決可否・キーワードなど）へ整形できます。

### 事前準備（認証情報の設定）

LLM 呼び出しには AWS の認証情報が必要です。次の 4 項目を設定します。

- AWS Access Key
- AWS Secret Key
- AWS Session Token（一時認証情報を使う場合のみ）
- AWS Region

設定内容は `config/bedrock_config.json` に保存されます
（**AWS認証情報を含むため Git 管理対象外**にしています）。
あわせて使用する**モデルID**（Bedrock の推論プロファイルID）と最大出力トークン数も設定します。

> モデルIDは利用するアカウント・リージョンで有効なものを指定してください。
> Claude 4.5 世代は APAC(`apac.`) プロファイルが無いリージョン/アカウントが多く、
> その場合は `global.` プロファイルを使います（既定値）。
> 例: `global.anthropic.claude-sonnet-4-5-20250929-v1:0` /
> `us.anthropic.claude-sonnet-4-5-20250929-v1:0`

### プロンプトのカテゴリ編集

分析プロンプトのうち「**## 9. category**」の分類一覧は
`config/categories.json` に保存され、GUI の「⚙️ 設定」画面から編集できます
（1 行に 1 つ）。それ以外のプロンプト本文は固定です。

### GUI での利用

1. 「⚙️ 設定」（画面右上）で AWS 認証情報・モデルID・カテゴリを設定して保存
2. 通常どおり文字起こしを実行（**話者分離あり**を推奨）
3. 結果カードの「🤖 **LLMで整形JSON出力**」をクリック
4. 整形された JSON が表示され、`.json` または `.csv`（📑 **CSVをダウンロード**）で
   ダウンロードできます

> 認証情報は画面では末尾4文字以外をマスク表示し、入力欄を空のまま保存すると
> 既存の値を維持します。

### JSON を CSV に変換する

整形 JSON は **問い合わせ 1 件 = 1 行**の CSV に変換できます。Excel でそのまま
開けるよう UTF-8（BOM 付き）で出力します。

画面上部の「📑 **JSON → CSV 変換**」カードは**文字起こしをしていなくても単独で
利用可能**で、次のいずれの形式の JSON にも対応します。

- 1 ファイル分（`{"inquiries":[…]}`）
- 複数まとめ（`{"a.wav":{…}, "b.wav":{…}}`、`分析まとめ.json` の形式）
- 問い合わせの配列（`[{…}, {…}]`）

使い方は、JSON を**テキスト欄に貼り付ける**か「📂 **JSONファイルを読み込む**」で
保存済みファイルを開き、「📑 **CSVに変換してダウンロード**」を押すだけです。
LLM 整形を実行済みのときは、各 JSON 表示横の「📑 **CSVをダウンロード**」や
変換カードの「↥ **直近のLLM整形結果を読み込む**」からも変換できます。

> 複数まとめの JSON を変換すると先頭に「ファイル名」列が付きます。`actions_taken`
> や `keywords` などの配列項目は ` / ` で連結してセルに収めます。

### 自分で LLM に投げたいとき（プロンプトのコピー）

Bedrock を使わず、**手元の任意の LLM（ChatGPT 等）に自分で貼り付けて使いたい**
場合は、結果カードの「📝 **LLM投入用プロンプトをコピー**」をクリックします。
設定済みのカテゴリ一覧と話者分離テキストを差し込んだ**プロンプト全文**が
クリップボードにコピーされるので、そのまま LLM に貼り付けてください
（この機能は AWS 認証情報なしで利用できます）。

### 複数ファイルの一括処理

複数の `.wav` をまとめて処理した場合に表示される「完了ファイル一覧」カードに、
全ファイルをまとめて扱うボタンが表示されます。

- 「📝 **全ファイルのプロンプトをまとめてコピー**」… 各ファイルのプロンプト全文を
  `################ ファイル名 ################` 区切りで連結してコピー（AWS認証情報なしで利用可）
- 「🤖 **全ファイルをLLM整形(JSON一括DL)**」… 各ファイルを順番に LLM 整形し、
  **ファイル名をキーにした 1 つの JSON**（`{ "a.wav": {...}, "b.wav": {...} }`）として
  `分析まとめ.json` をダウンロード
- 「📑 **直近のLLM整形結果をCSVでDL**」… 上記の一括整形の結果を、ファイル名列付きの
  CSV（`分析まとめ.csv`）としてまとめてダウンロード（整形実行後に表示）

> 一括整形は 1 ファイルずつ順番に Bedrock を呼び出します。失敗したファイルは
> JSON 内にそのファイルの `error` を記録して処理を継続します。同名ファイルが
> ある場合はキーを `名前 (2)` のように自動で区別します。

### CLI での利用

`--analyze` を付けると、文字起こしと同時に `<名前>.analysis.json` を出力します。

```bat
venv\Scripts\python.exe cli.py --diarize --analyze
```

実行前に `config/bedrock_config.json` を設定しておく必要があります
（GUI の「⚙️ 設定」画面から作成すると簡単です）。

## 設定（環境変数）

| 変数 | 既定 | 説明 |
|------|------|------|
| `WHISPER_MODEL` | large-v3 | 既定モデル |
| `WHISPER_DEVICE` | cpu | `cpu` / `cuda` |
| `WHISPER_COMPUTE_TYPE` | int8 | `int8` / `float16` 等 |
| `DIARIZE_THRESHOLD` | 0.68 | 話者自動推定のコサイン距離しきい値 |
| `PORT` | 5000 | サーバのポート番号 |

## ファイル構成

```
音声文字起こし/
├─ app.py                … Flask バックエンド（ジョブ管理・進捗API・設定/分析API）
├─ transcribe_core.py    … 文字起こしの共通コア（GUI / CLI から利用）
├─ llm_analyze.py        … LLM(Bedrock)問い合わせ分析の共通コア（GUI / CLI から利用）
├─ cli.py                … CLI モード（フォルダ一括処理）
├─ diarization.py        … 話者分離（SpeechBrain + scikit-learn）
├─ index.html            … GUI（HTML / JavaScript）
├─ config/               … LLM設定（bedrock_config.json / categories.json、Git管理対象外）
├─ requirements.txt      … 依存ライブラリ
├─ setup.bat             … 初回セットアップ
├─ start.bat             … GUI 起動
├─ transcribe_folder.bat … CLI モード起動（INPUT → OUTPUT）
└─ README.md
```
